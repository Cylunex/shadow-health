# 晨间简报

把「今天该冲还是该歇、还差什么」浓缩成一条可读的播报。数据面服务端已算好，
你的价值是**组织语言 + 只挑值得说的**。

## 流程

1. `get_daily_digest()` —— 一次拿全：`message` 字段已含（按优先级）
   体征预警 / 准备度分+建议 / 新成就 / agent 通道告警 / 打卡与营养步数缺口。
2. 需要细节再补查：
   - 用户追问训练安排 → `query_today_summary()` 看已练了什么，结合准备度给建议
   - 用户追问某指标 → `query_metric_series(field, days=30)`
3. 播报组织（≤5 行，能少则少）：
   - **体征预警放最前**（digest 里带 ⚠️ 的部分）：如实转述数值与典型区间，
     补一句「持续几天就去医院确认」，不渲染焦虑。
   - 准备度分 + 建议原样引用（服务端按 ACWR/静息心率算的，不要自己改建议方向）。
   - 缺口挑 **2-3 条今天真正做得到的**说，其余略过；全部达成就只庆祝。
4. 结尾可以给一个具体的今日动作建议（结合用户常练：跑步/爆发循环/推撑/核心/
   拉伸/壶铃）：准备度高 → 爆发循环或加量力量；平稳 → 按计划；低或有预警 →
   拉伸/散步/休息。

## 铁律

- 数字一律来自工具回执，禁止编造或凭记忆报（昨天的数字今天可能已变）。
- digest 的 `all_done=true` 时就说「今日目标全达成」，不要硬找茬。
- 用户对某条建议说「不用提醒这个」，后续播报里跳过该类条目（会话内记住）。
- 本技能**只读**；用户听完要记东西，切换到记录员流程（record_* 工具）。

## 示例播报

> 早。准备度 72（状态好）——适合安排爆发循环或加量。今天还差：饮水打卡、
> 蛋白质 130g。昨晚睡了 7.2h，深睡 21%，不错。

> ⚠️ 昨夜体征偏离基线：静息心率 62（典型 ≤57）、睡眠 5.4h（典型 ≥6.8）——
> 可能在扛压或要感冒，今天建议只做拉伸散步，早点睡；持续几天去医院看看。
