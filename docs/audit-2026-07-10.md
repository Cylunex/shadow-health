## 🔴 高危

### H1 ☑ 月报「有氧达标周」提前冻结，跨月末周必然误判
- 位置：`app/routers/report.py:467`（`_ensure_last_month` / `_get_or_create_month`）
- 场景：口径允许周一落在本月的 ISO 周跨入下月（fetch_end 向后多取 6 天），但生成条件只要求
  「月已结束」——下月 1 号首次打开月报 tab 即落库冻结。例：2026-06 含周一 6-29（该周到 7-5
  结束），7-1 访问 → 六月快照落库，该周有氧只算到 7-1，7-2~7-5 补的训练永不计入，
  月报永远显示「4 / 5 周」（快照只读、无重算机制）。每个末周跨月的月份都有 1~6 天触发窗口。
- 修复：生成条件改为 `_month_mondays(ms)[-1] + timedelta(days=7) <= today_local()`
  才冻结快照；末周未结束期间访问详情可现算不落库。

### H2 ☑ Keep 导入跨源去重漏 samsung_direct，运动双计
- 位置：`app/importers/keep_file.py:52`（`DUP_SOURCES`）
- 场景：手表运动经直读通道写入 `workout_logs(source='samsung_direct')` 后再导 Keep 文件：
  `_import_exercise` 的去重只查 `('samsung_zip','health_connect')`，同一次跑步
  （started_at 相差 ≤3 分钟）不命中 `_dup_near`，以 `source='keep'` 再插一条 →
  训练列表、周有氧、热量全部双计。直读现已是主通道，实际影响最大。
- 修复：`DUP_SOURCES` 加 `'samsung_direct'`（三元组即可，dup_ts 查询自动生效）。

### H3 ☑ Android 壳缺 WebChromeClient：确认框全部静默取消、文件上传打不开
- 位置：`android/.../MainActivity.java:97`
- 场景：只调了 `setWebViewClient`，未设 `WebChromeClient` → WebView 里 `window.confirm()`
  一律返回 false：壳内点任何 hx-confirm 删除/放弃动作（饮食行、训练行、食物、计划）
  毫无反应；且缺 `onShowFileChooser`，`<input type="file">` 打不开——**壳内拍照记餐、
  导入 zip 全部不可用**。
- 修复：`setWebChromeClient(new WebChromeClient(){...})`（alert/confirm 自动走系统对话框），
  覆写 `onShowFileChooser` 用 ACTION_GET_CONTENT 回传 Uri。

## 🟠 中危

### M1 ☑ zip 导入当天，直读 daily 数据被水位线冻结
- 位置：`app/routers/ingest.py:834`（rec_ts 取当日 00:00）、`:964`（水位线比较）；
  `samsung_zip.py:740-746`（zip 导入把水位线写成导出时刻）
- 场景：zip 导完当天（水位线≈导出时刻如 14:12），直读推送当日 daily 聚合的
  rec_ts=00:00 恒 ≤ 水位线 → 全部 skipped。当天步数/心率冻结在 zip 半日快照，
  今日三环、步数目标、20:30 提醒 digest 全部基于过期值，次日才自愈；
  且步数增长使 ext_id 哈希变化，每小时新插一条注定 skipped 的 import_raw 垃圾行。
- 修复：daily 类记录与水位线比较用当日结束时间（次日零点），或按
  `d > watermark.date()` 才 skip（等于当天放行走 SET 覆盖，幂等无副作用）。

### M2 ☑ zip 重导的 sleep_hours 回填绕过跨源去重，覆盖直读值
- 位置：`app/importers/samsung_zip.py:481`（`import_sleep` 回填）
- 场景：直读运行一段时间后重导覆盖同期的 zip：回填用 zip 自身 `hours_by_wake` 求和直接
  `autofill_fields` 覆盖（字段已登记 autofilled 即允许改写），而睡眠图/日报走
  `services/sleep` 优先取 samsung_direct——两值不一致时指标页/LLM 上下文与睡眠图互相矛盾，
  且 zip 水位线让直读跳过历史，错值不自愈。ingest 的 HC/直读通道都已改用
  `sleep.total_sleep_min` 同口径，唯独 zip 导入器漏了。
- 修复：`import_sleep` 对每个受影响 wake_date 调 `services.sleep.total_sleep_min(db, d)` 回填。

### M3 ☑ LLM 快照对 weekly 习惯口径算错
- 位置：`app/services/llm.py:190`
- 场景：habits 模块 weekly 口径 = 本周 done_count 求和 ≥ target；build_context 却按
  「单日 done_count ≥ 周目标」计行数并标注「X 周」。周目标 2 次分两天完成的习惯
  （seed 自带「深海鱼×2」「称重×2」）在 AI 上下文里永远是「0 周」，AI 报告据此下错结论。
- 修复：weekly 习惯按 `_week_start` 分组求和后比较（复用 habits.`_week_sums` 口径）
  统计达标周数；daily 保持现逻辑。

### M4 ☑ 周报/月报列表存在永久空洞
- 位置：`app/routers/report.py:437`（`_ensure_last_month`）、`review.py:184`（`_ensure_last_week`）
- 场景：只惰性生成「上一完整期」。导入一年历史数据后月报列表只有上个月一份；
  整月未访问则该月永远缺席（只能手敲 URL 生成）；周报同理，外出两周留一个不可达的洞。
- 修复：列表构建时从最早有数据的周期回填缺口逐一 `_get_or_create`；
  或详情页加上一期/下一期导航按需生成。

### M5 ☑ 餐次照片上传失败零反馈
- 位置：`app/routers/diet.py:636` + `templates/fragments/diet_day.html:48`
- 场景：超 15MB / 格式不符 / meal 非法 → 抛 400/413；上传表单 `hx-swap="none"`、
  htmx 对 4xx 不 swap、全站无 `htmx:responseError` 处理器 → 界面只闪一下 pulse，
  用户以为传上了。对比同页 AI 识别失败有红色错误条，上传路径漏了错误通道。
- 修复：照 `diet_photo_analyze` 模式失败返回 200 + 错误片段；或 base.html 加全局
  `htmx:responseError` toast（与 UX-3 全局 toast 合并做）。

### M6 ☑ 今日三环不监听 habit-changed/workout-changed
- 位置：`templates/today.html:18`
- 场景：同页打卡/按模板完成后，汇总条和计划卡都刷新了，顶部三环仍是加载时的静态 SVG，
  与正下方数据互相矛盾，需手动刷新。
- 修复：三环抽成 `/fragments/today/rings` 片段，容器加
  `hx-trigger="habit-changed from:body, workout-changed from:body"`。

### M7 ☑ 启动计划选未来日期被静默钳制为今天
- 位置：`app/routers/workout.py:609`（`plan_enroll` 复用 `_parse_date` 的 min(…,today) 钳制）
- 场景：选下周一启动 → start_date 被改为今天，计划卡立即显示「第 1 周第 1 天」，
  周次全程错位；`not_started` 分支与「将于 X 开始」UI 成死代码（钳制是 8bab2bd
  为训练日志加的，误伤计划报名）。
- 修复：`plan_enroll` 单独解析 start_date（解析失败才回退今天），不做未来钳制。

### M8 ☑ SW 精缓存可能固化旧静态资源
- 位置：`static/sw.js:26`（install addAll）、`:56`（运行时回源）
- 场景：Starlette StaticFiles 不发 Cache-Control，浏览器对 /static/* 走启发式缓存；
  升 SW_VERSION 后新 SW install 时 addAll 可能从 HTTP 缓存拿到旧 app.css 存入新版本缓存，
  整个新版本周期返回旧 CSS——「改 SW_VERSION 即整体失效」的注释前提不成立。
- 修复：`addAll` 改 `PRECACHE.map(u => new Request(u, {cache: 'reload'}))`；
  运行时回源 fetch 用 `{cache: 'no-cache'}`（或给 /static 响应加 Cache-Control）。

### M9 ☑ NAS 网关 BLE 静默停摆无看门狗
- 位置：`gateway/miscale_listener.py:185`
- 场景：宿主机 bluetoothd 重启/USB 蓝牙棒重插后 discovery 会话丢失，bleak 不重建扫描
  也不抛异常——进程活着，Docker `restart: unless-stopped` 永不触发，称重数据静默丢失。
- 修复：看门狗——记录最近收到广播/扫描活跃时间，超阈值主动退出交给 Docker 重启；
  或周期性重建 BleakScanner；至少在静默时打日志。

# 功能建议（按价值÷成本排序）

### F1 ✅ 餐次一键复制（small）
单人饮食高度重复，一个按钮替代每天多次搜索。新增 `POST /diet/meals/copy`（form: d + meal）：
找最近一个早于 d 且该餐次有记录的日子，整餐复制到当天（food_id 行按现行食物库重算营养，
free_text 原样），响应带 `HX-Trigger: diet-changed` 即自动刷新。UI 在 diet_day.html
餐次分组头（当日该餐为空时）显示「复制上次早餐」。进阶：「存为常用餐」组合存 app_settings。

### F2 ✅ 能量收支缺口卡（medium）
摄入（diet_logs）− (BMR（body_metrics.bmr_kcal，体脂秤回填）+ 活动消耗
（daily_activity.active_kcal））三项已全在库里。`_summary_ctx` 加缺口计算，
diet_summary.html 加一行「今日缺口 −420 kcal（减脂建议 300–500）」——饮食页/今日面板/日报
三处自动共享；周/月快照加周均缺口。身高性别可做 Mifflin-St Jeor 兜底估算 BMR。

### F3 ✅ auto_rule 派生字段扩展（medium）
「23点前睡」「称重」「训练」「记录饮食」的证据都在库里。`_rule_value` 白名单扩展：
`workout_min`（当日训练分钟求和）、`diet_count`（当日饮食笔数）、`bedtime_ok`
（当夜最早 start_at 早于 23 点，跨午夜用「距中午分钟数」比较）。注意：weekly 习惯的
`_apply_auto_rules` 语义要改成每天最多 +1（现在一天直接插满 target）；
与 L4（撤销否决标记）配合落地；seed 同步配规则、补测试。

### F4 ✅ 习惯补记昨天/指定日期（medium）
习惯是唯一无法补记的数据，漏一天断连击。toggle/increment/decrement 三端点接受可选
表单参数 d（≤今天、限最近 N 天），habits_today_item.html 按钮 hx-vals 带日期；
入口放日报页打卡清单——翻到昨天时圆圈变可点补卡按钮，日报页顺势成为补记中心。

### F5 ✅ 循环训练引导播放器（medium）
动作库 × 计时器 × 「常用全身循环」计划打通（Keep 式跟练）。/workout/timer 接受
`exercises=id1,id2…`，工作相位大字显示当前动作名+要领，每轮自动切下一动作；
入口在动作分类页「用这组开练」和计划卡「跟练」；结束接 UX-1 一键记入。纯本地，符合断网原则。

### F6 ✅ 跑步配速 + 趋势图（medium）
distance_km + duration_min 已齐备。①半天：训练行/日详情对有距离时长的记录显示配速 min/km；
②metrics `_CHART_METRICS` 加「跑步」：周跑量柱状 + 平均配速折线，复用现有 Chart.js 管线。

### F7 ✅ 一键全量备份含照片（medium，建议先做部署侧）
照片目前不在任何备份里。①部署侧几行脚本：backup 服务 pg_dump 后追加 photos 目录 tar；
②应用侧 `GET /settings/backup` 流式 zip（逐表 CSV + photo_dir），设置页加入口。

### F8 ✅ 力量组次明细 + PR + 进阶提示（large，handover 候选池方向）
手动表单加可折叠「动作明细」（动作×组×次×重量，Alpine 动态行），写 detail JSONB
（无需迁移）；行/日详情渲染明细；新 services/pr.py 扫描 PR（最大次数/重量/单组容量）
展示在动作卡与成就区；同 progression_group 达标时提示「可进阶下一 level」。

# 使用优化（除注明外均 small）

### U1 ✅ 计时器练完一键记入训练
timer done 相位加「记入训练」：hx-post 现有 POST /workout/logs，Alpine 算好
duration_min=轮数×(work+rest)、session_type=间歇训练，成功变「已记录 ✓」。

### U2 ✅ 今日页「拍照记一餐」直达
快速记饮食卡加相机按钮（`<input type=file accept=image/* capture=environment>`），
hx-post 现有 /diet/photos，meal 服务端按时间预选（允许 meal 为空回退 `_default_meal`）。

### U3 ✅ 全局 toast（成功确认 + 错误兜底）
base.html 加底部 toast 容器 + ~15 行 JS：监听 HX-Trigger JSON 的 toast 事件
（diet_quick/photo_upload 等附带「已记录 早餐·鸡胸肉 150g」），同时挂
htmx:responseError 红色兜底——与 M5 一起做，一处改动全站受益。

### U4 ✅ 「按模板完成」自动带上次同类型时长/RPE
log_create 中 from_template 且 duration 为空时，回填最近一条同 session_type 的
duration_min/RPE（detail 标注 auto_filled）——三环训练环和周负荷零点击变准，与 L3 互补。

### U5 ✅ 常吃 chips 按当前餐次时段排序
`_chips_ctx` 优先取 `_default_meal()` 餐次下的频次 top，不足 8 个用全局补齐。

### U6 ✅ 食物搜索联想按使用频次排序
`diet_food_search` 现按名字长度排序；改 LEFT JOIN 近 90 天使用次数，
按（次数 desc, 长度 asc），命中过的附「常吃」徽标。

### U7 ✅ 提醒 digest 先跑 auto_rule + 补饮食/称重缺口
digest 先调 `_apply_auto_rules` 再算 pending（不再把已达标的「步数达标」误报）；
补两类缺口：今日饮食 0 笔提示「还没记饮食」、周后半「称重」未达标提示。

### U8 ✅ 饮食页/日报页日期点开日历跳转
翻天胶囊中间换透明覆盖的 `<input type=date max=今天>`，change 跳转——补记不再连点 ‹，
也缓解日报列表只有 14 天的入口问题。

### U9 ✅ chips/tab 切换加载态防连点
Tailwind 给 .chip 加 `.htmx-request` 时 opacity+pulse 规则；报告中心 tab（周报首次聚合最慢）、
指标 chips、热力图月份按钮补 `hx-disabled-elt="this"`。注意重建 CSS 后升 SW_VERSION。

### U10 ✅ AI 分析改后台任务 + 轮询（medium）
1-2 分钟同步请求在 WebView 锁屏即作废。仿导入中心 job 模式：入库任务立即返回轮询片段，
后台线程完成写缓存，前端 3s 轮询——AI 功能在手机上从基本不可用变成可用。
